# 歴史因果関係マップ (TOC結合点版) 構築ガイドライン

このドキュメントは、歴史の因果関係をTOC（Theory of Constraints：制約条件の理論）の思考プロセスを用いて分析し、React Flow用のデータとして出力するための共通ルールです。以降のセッションで「歴史マップの作成・拡張」を求められた際は、このガイドラインを自動的に読み込んで従ってください。

## 1. ノードのデザインルール（画面表示とメタデータ）
- **プレーンなテキスト表示:** ノードの画面上の表示文字（`data.label`）には、**「【UDE:圧迫】要素名」のようなメタデータの修飾・文字装飾を一切付けてはならない**。純粋な出来事や状態の名称のみとする。（例：〇「ゲルマン人の大移動」 / ×「【事象】ゲルマン人の大移動」）
- **メタデータは裏側のデータ属性へ:** TOC思考プロセスの属性（原因、結果など）は、データ構造の `data.type` に持たせること。

## 2. TOCデータ構造 (JSONメタデータ) の定義
ノードを定義する際には、必ず以下の `data.type` を設定し、対応する色彩・スタイルを適用する。
- `root_cause`: 根本原因（気候変動など）
- `event`: 事象・出来事
- `ude`: 望ましくない結果 (Undesirable Effect)
- `action`: 行動
- `context`: 前提条件・状態（例：ローマ帝国の兵力不足）
- `injection`: 解決策・対策（インジェクション）
- `nbr`: 対策による副作用 (Negative Branch)
- `result`: 最終結果

## 3. AND / OR の論理結合ルール（Logic Nodes）
複数の原因が絡む場合（遠因と近因など）は、事象同士を直接矢印で結ばず、必ず**論理結合点（Logic Node）**を経由させること。
- **AND結合:** `type: 'logic', data: { operator: 'AND', symbol: '＊' }`
- **OR結合:** `type: 'logic', data: { operator: 'OR', symbol: '＋' }`
- **矢印のテキスト排除:** 矢印（エッジ）には「THEN」などの補足テキスト（ラベル）は付けない。すべて単なる有向エッジ（矢印）とし、線の色（strokeColor）で論理的な意味の繋がりを表現する。

## 4. 「遠因」や「実現しなかったIF（失敗ルート）」の表現手法
歴史上「なぜAではなくBになったのか」という消去法の必然性、または「ある遠因がなぜ歴史を動かしたのか」という間接的な影響を示す場合は、以下の構造（IF/THENロジック）で表現する。

1. **試みの発生**: 元々の動機によって本来目指したかった事象を描画（例：豊かな東方（中国）への試み）
2. **前提条件との衝突**: その行動を阻む強固な前提条件を描画（例：万里の長城、強固な防衛網）
3. **AND結合による失敗**: (1)の試みと(2)の前提条件を `AND(＊)` で結合し、「〜への侵入失敗・断念」という結果ノードを導き出す。
4. **必然の別ルート発動**: その「失敗・断念した結果」と、「根本的な生存危機（元々の動機）」を再び別の `AND(＊)` で結合することで、「結果的に別の行動（例：西進）を取らざるを得なかった」という本筋への合流として表現する。
